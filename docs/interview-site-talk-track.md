# Interview Site Talk Track

這份筆記是給你在面試時搭配個人網站使用的講稿底稿。

使用原則：
- 每一頁只講一個核心訊息
- 不要逐字朗讀頁面文字
- 先講結論，再補例子
- 如果面試官有興趣，再往下一頁或展開細節

---

## 1. Intro

### 這一頁要讓面試官記住什麼

- 我是有後端基礎、也有全端實作經驗的軟體工程師
- 我最近最擅長的是把複雜 workflow、整合需求、產品規則變成可用的系統
- 我的強項不是單一技術點，而是把模糊需求整理成可以落地的產品功能

### 建議講法

「我現在比較穩定的定位是 Software Engineer / Product Engineer。  
我有後端基礎，也做過全端產品開發，最近主要用 Node.js / TypeScript。  
如果要用一句話講我在做什麼，我會說我是把複雜 workflow、整合需求和產品規則整理成可用系統的人。」

### 如果對方追問，可以補什麼

- 現在的工作有 React dApp、NestJS API、SDK、資料流程、AI workflow
- 過去也做過 C# / .NET 後端維護與重構
- 所以你不是只懂某一層，而是能跨前後端把功能接起來

---

## 2. About

### 這一頁要讓面試官記住什麼

- 研究背景不是過去式，而是現在做工程的方法來源
- 你的做事方式是先觀察、再整理、再落地
- 你會在意真實使用情境，不只是把功能做出來

### 建議講法

「我以前做的是生物醫學研究，所以我很習慣先觀察問題、整理脈絡、測試假設，再決定怎麼做。  
這個習慣其實一直延續到現在做工程。  
所以我做產品時，不會只看功能能不能做出來，也會看資料怎麼流、使用者怎麼理解、這個流程是不是真的能被使用。」

### 如果對方追問，可以補什麼

- 為什麼你對 domain-heavy 或 workflow-heavy 的問題特別有感
- 你面對陌生領域時，通常怎麼快速上手
- 研究背景如何幫助你做跨團隊溝通與文件整理

---

## 3. Work

### 這一頁要讓面試官記住什麼

- 你最近的工作經驗是有真實產品脈絡的，不是純練習專案
- LAVARAGE 代表你的最新能力輪廓
- TitanSoft 代表你的後端基礎與維護、重構、整合經驗

### 建議講法

「最近一段工作經歷主要是 LAVARAGE。  
我在跨國、全英文環境裡參與 Web3 / DeFi 產品開發，工作內容橫跨 React dApp、NestJS API、SDK、資料流程，還有後來獨立設計的 AI support workflow。  
再往前一份是 TitanSoft，主要是 C# / .NET 後端系統維護、重構、第三方整合和測試補強。  
所以如果把我的工作經歷濃縮來看，一條線是產品與 workflow，另一條線是後端、整合與系統穩定性。」

### LAVARAGE 可以優先講的 3 條線

- Referral system：很適合講跨前後端協作與完整功能落地
- Transaction flow / integration：很適合講複雜流程、穩定性、外部整合
- Lavi：很適合講 ownership、AI workflow、模糊需求落地

### 如果時間有限，建議只挑一條展開

- 面試偏產品工程：先講 referral system 或 Lavi
- 面試偏後端：先講 transaction flow / integration
- 面試偏 AI / automation：先講 Lavi

## 4. Strengths

### 這一頁要讓面試官記住什麼

- 你不是「什麼都會一點」的全端，而是有一套很清楚的方法和強項
- 你的 value 在於 workflow thinking、backend foundation、domain learning

### 建議講法

「我覺得我現在能帶給團隊的價值，不是某個單點技術，而是三件事。  
第一是 workflow thinking，我很擅長先拆流程、找邊界，再把複雜需求落成可運作的產品功能。  
第二是 backend foundation，我做過 API、資料流程、整合，也有 C# / .NET 跟 Node.js / TypeScript 的經驗。  
第三是 domain learning，我對陌生領域的上手速度算快，也能把模糊問題整理成可執行方案。」

### 如果對方追問，可以補什麼

- 為什麼你會把自己放在 Product Engineer 的脈絡
- 你和「純前端」或「純後端」工程師最大的差異
- 你喜歡什麼樣的問題型態

## 5. Selected Work

### 這一頁要讓面試官記住什麼

- 你的 side projects 不是作品堆疊，而是能力地圖
- 每個作品都對應一種能力切面
- 面試時不要每個都講，挑 2 個主力 + 1 個備用就夠

### 建議講法

「這一頁我比較把它當成代表作品區，而不是單純作品集。  
我通常會挑其中兩個主講，因為它們最能代表我現在的能力輪廓。」

### 建議主講順序

#### TrackVest

- 代表：產品思維、全端架構、資料模型、可持續迭代
- 建議講法：

「TrackVest 是我長期在做的個人記帳與投資系統。  
它對我來說最重要的地方不是做了一個 CRUD app，而是我自己去定義帳戶、資產、交易、總帳、CSV 匯入這些模組和資料流，這很能代表我怎麼做產品系統設計。」

#### JD Saver

- 代表：workflow automation、integration、human-in-the-loop system
- 建議講法：

「JD Saver 對外看起來像一個 Chrome extension，但我現在比較把它看成更大 job search workflow system 的入口。  
它的價值不只是把資料寫進 Google Sheet，而是把求職資料收集這件事整理成可重複使用的 workflow。」

#### Lavi

- 代表：AI workflow、ownership、系統邊界、安全規則
- 建議講法：

「Lavi 是我最近很能代表自己的案例。  
因為它不是單純接一個 LLM API，而是要把即時回覆、背景同步、判重、issue sync、guardrails 全部整理成完整 workflow。」

### 可作為補充的作品

- Split-Bill
  - 適合講產品感、前端體驗、從真實需求出發
- Board Game Record
  - 適合講早期全端能力與資料庫 / 驗證 / 管理流程

### 面試時建議怎麼選

- 偏產品工程：TrackVest + JD Saver
- 偏後端 / 平台：TrackVest + Lavi
- 偏 AI / automation：Lavi + JD Saver
- 偏前端 / 使用流程：Split-Bill + TrackVest

## 面試時的整體節奏

### 3 分鐘快速版

1. Intro：你是什麼類型的工程師
2. About：研究背景如何影響你做工程
3. Work：最近做過什麼
4. Selected Work：挑 1 到 2 個案例

### 5 到 8 分鐘完整版

1. Intro
2. About
3. Work
4. Strengths
5. Selected Work
